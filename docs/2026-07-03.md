# RawSight 开发日志 — 2026-07-03

## 版本 v1.0.2

### 修复

- **白平衡不生效**：`CameraService.applyCameraState()` 缺少 `COLOR_CORRECTION_MODE_TRANSFORM_MATRIX`，导致手动 WB 的 `COLOR_CORRECTION_GAINS` 不生效。已补充。
- **预览轻微拉伸**：`applyCropTransform` 中 `setRectToRect(FILL)` 后多余的 `postScale` 导致二次缩放。已移除，仅保留 `setRectToRect`。
- **EV 不生效**：`CONTROL_AE_EXPOSURE_COMPENSATION` 仅在 `CONTROL_AE_MODE != OFF` 时生效。当 ISO 或快门为 MANUAL 时，AE 关闭，EV 补偿无效。当前处理：选 EV 标签时若处于手动曝光则显示黄色提示 "EV needs AUTO exposure"。

### 新增

- **离散档位滑块**：ISO/SPD/WB/EV/FOC 全部改为整数索引滑块，只在预设值之间跳转，避免低 ISO 区域精度不足。每个参数有独立步骤数组。
- **对焦控制外移**：FOC 标签加入底部滑块栏。支持 AF/MF 切换，MF 模式下可拖滑块调节对焦距离 (0.0-1.0，7 个离散档位)。
- **版本号递增**：v1.0.1 → v1.0.2。APK 文件名随版本号变化。

### 设计决策

- **离散 vs 连续滑块**：对于 ISO/SPD/WB，标准相机只有预设档位（如 100/200/400），无极调节在此没有实际意义且会让用户困惑。离散滑块更接近真实相机旋钮体验。
- **EV 与手动曝光**：EV 补偿本质上是对自动曝光的偏移。当用户手动控制 ISO/快门时，已经直接决定了曝光，EV 补偿不再有意义。当前提示用户切回 AUTO 曝光。

## 版本 v1.0.1

### 修复

- **预览挤在左边/缩小到 1/8**：`Modifier.aspectRatio()` 与 Matrix 变换冲突。最终方案：去掉 `aspectRatio`，用 `fillMaxSize()` + `addOnLayoutChangeListener` 获取实际 View 尺寸 → `setRectToRect(FILL)` 矩阵做 CENTER_CROP。
- **镜头重复出现两个 2.6x**：`LensManager` 枚举所有 cameraId 时包含 LOGICAL_MULTI_CAMERA（虚拟逻辑镜头，组合多个物理镜头），导致同一焦距出现两份。新增过滤逻辑 + 按焦距四舍五入去重。
- **手动参数无法调整**：`onToggle` 中传给 `onParam` 的值为 `null`，`updateParameter` 使用 `?:` 保留了旧值。修复：切换时传入当前值。

### 新增

- **前置摄像头**：`LensManager.enumerateAll()` 返回前后全部非逻辑镜头。UI 右上角分 REAR/FRONT 两组显示，点 F 切换到前置。
- **滑块参数控制**：替换旧的点按循环 Chip，改为 Material3 `Slider`。选择参数标签后在底部拖动调节。
- **APK 版本控制**：`applicationVariants` 配置 `outputFileName = "RawSight-v${versionName}.apk"`，不覆写旧版本。

### 已知问题（v1.0.1 时）

- 滑块为连续值，ISO 64-800 区间太窄，精度不足（v1.0.2 已修复为离散）。
- 白平衡拖动无变化（v1.0.2 已修复）。
- EV 拖动无变化（v1.0.2 已加说明）。
