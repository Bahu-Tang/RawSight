✅ OPENCODE PROMPT（Demo落地版·极致摄影辅助系统）
🧠 ROLE

你是一个Android高级Camera与计算摄影工程师，需要在 OPPO Find X8（Android 13） 上实现一个“AI摄影决策辅助系统 Demo”。

目标不是滤镜或后期，而是：

在拍摄前提供最优构图与曝光决策，并尽可能保留原始光学数据（RAW优先）。

系统必须优先保证 Camera稳定性与可运行性。

🎯 PROJECT GOAL（核心目标）

实现一个可运行的摄影App Demo，包含：

1. 纯净相机采集（核心）
使用 Camera2 或 CameraX
单 Camera session
输出：
Preview（低处理）
JPEG（必须）
RAW DNG（如果设备支持）
必须要求：
禁用所有AI场景增强
禁用HDR自动优化（如可控）
禁用美颜/滤镜
使用 MANUAL exposure control（ISO / shutter / WB）
2. 专业手动控制模块

必须实现：

ISO：64–6400
快门：1/8000s–30s
白平衡：2500K–7500K
手动对焦（MF）
EV补偿

支持每个参数：

AUTO / MANUAL 独立切换
3. AI摄影决策模块（轻量，不影响拍摄）
输入：
Camera preview frame（降采样）
每秒最多 2 FPS 分析
输出（仅建议，不控制相机）：
sceneType（人像 / 风景 / 夜景 / 微距）
subject bounding box
composition suggestion：
move left/right/up/down
tilt correction suggestion
exposure suggestion：
recommended ISO / shutter (仅显示建议值)
模型要求：
使用 TFLite（MobileNet / EfficientNet-lite / YOLO-nano）
必须异步运行
不允许阻塞 Camera pipeline
4. 构图辅助UI（2D，不使用ARCore）

实现 overlay：

三分法网格
主体检测框
构图偏移箭头（左/右/上/下）
水平仪（IMU）
UI规则：
只使用 Canvas / OpenGL overlay
不允许 ARCore / SLAM / 3D重建
5. 拍摄决策提示系统（核心体验）

在用户按下快门前：

系统计算：

曝光是否过曝/欠曝
主体是否偏离构图中心
是否存在抖动风险（IMU）
对焦是否稳定

然后输出：

状态之一：
✅ “Optimal Shot Ready”
⚠ “建议调整构图”
⚠ “光线不足，建议降低快门或提高ISO”
❌ “当前不建议拍摄（严重抖动/过曝风险）”
🧱 ARCHITECTURE（必须严格拆分）

项目必须分为以下模块：

1. CameraService（核心）

职责：

管理 Camera2 lifecycle
打开/关闭 camera session
镜头切换（0.6x / 1x / 3x）
设置曝光 / ISO / WB

约束：

单 session
所有输出共享同一 preview stream
2. CaptureController

职责：

拍摄 JPEG
保存 RAW（DNG）
文件管理
3. AIAnalyzer（轻量）

职责：

低频帧分析（≤2 FPS）
输出构图 + 场景 + 曝光建议

约束：

必须异步线程
不得影响 preview FPS
4. UIOverlayController

职责：

绘制 grid
绘制 bbox
绘制箭头建议
绘制水平仪

禁止：

不做任何计算
不访问 Camera
5. SettingsEngine

职责：

管理 AUTO / MANUAL 状态
管理摄影参数状态机
6. DecisionEngine（新增核心）

职责：

汇总 AI + Camera metadata
输出拍摄建议状态
判断“是否适合拍摄”
⚙️ PERFORMANCE REQUIREMENTS
Preview ≥ 30 FPS
AI ≤ 2 FPS
内存 ≤ 600MB
Camera pipeline 不允许卡顿
AI异常不得影响拍摄功能
温控策略：高温自动关闭AI模块
❌ STRICTLY FORBIDDEN

必须遵守：

❌ 不使用 ARCore
❌ 不做3D建模或SLAM
❌ 不允许多 Camera session
❌ 不允许 AI 控制 Camera 参数（只能建议）
❌ 不允许高频 inference（>2 FPS）
